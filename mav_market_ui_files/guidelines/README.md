# `guidelines/`

Project guideline workspace for AI/development guardrails.

## Current content

- `Guidelines.md` currently contains a template scaffold ("add your own guidelines here").

## Intended use

Use `Guidelines.md` to capture team-specific generation rules, for example:

- UI constraints
- design-system tokens and component usage rules
- architecture do/don't patterns
- naming and interaction conventions

Keep guidance concise and actionable; avoid generic statements that do not affect implementation decisions.
